# 文档贡献指南

## 目标

保持文档树准确、双语路径镜像，并且严格基于仓库中当前已经实现的行为。

## 规则

1. 保持英文和中文页面路径镜像。
2. 尽量在同一次修改中同步更新两种语言。
3. 先记录当前行为，再明确标注未来规划或待实现能力。
4. 使用简短分节、直接表达和具体示例。
5. 文件移动后同步修复链接。

## 写作风格

- 优先使用主动语态。
- 删除无意义赘词。
- 多用便于快速扫描的列表。
- 只有在命令、配置或代码示例时才使用代码块。

## 当翻译暂时不同步时

如果你只能先更新一种语言，请在另一种语言页面顶部加一条简短说明，标明需要同步，不要让两份文档无提示地长期漂移。

## 建议流程

1. 先更新英文页面。
2. 用相同结构同步到中文页面。
3. 检查本地链接。
4. 如果文档树结构变化，同步更新 `docs/README.md`。
